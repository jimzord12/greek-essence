## 21. Implementation Phases

### Phase P0 — Bootstrap

Create the Next.js/TypeScript 6 app, configure Tailwind v4, shadcn/Base UI, `next-intl`, strict lint/type settings, local/preview/prototype environment variables, Vercel project, and Resend test domain.

### Phase P1 — Visual and content foundation

Implement local content schemas/loaders, locale routes, global layout, C-01–C-24, C-28–C-30, static pages, imagery pipeline, SEO metadata/sitemap/robots, responsive modes, accessibility baseline, and subtle motion system.

### Phase P2 — Conversion prototype

Implement C-25/C-27, four form steps, conditional fields, draft persistence/clear/expiry, client/server schemas, route handler, Resend emails, Gmail notification, acknowledgement templates, errors/retry, and direct-contact fallback.

### Phase P3 — Demo hardening

Complete Playwright flows, responsive/keyboard/zoom/reduced-motion checks, content/translation validation, email smoke tests, image/bundle optimization, custom domain, and client-demo rehearsal.

### Phase P4 — Owner review and decision

Use the prototype to review visual direction, content quality, mobile form experience, language behavior, trust presentation, and whether to fund a production MVP. Do not promote it to public commercial operation without §22 review.
