
# Part V — Spacing and Layout

## 9. Spacing Scale

Use a 4 px base grid with selected editorial increments.

| Token | Value | Typical use | Responsive behavior |
|---|---:|---|---|
| `space-0` | 0 | reset | fixed |
| `space-1` | 4 px | icon/label micro gap | fixed |
| `space-2` | 8 px | compact control gap | fixed |
| `space-3` | 12 px | helper text, small stacks | fixed |
| `space-4` | 16 px | default internal spacing | fixed |
| `space-5` | 20 px | control/card spacing | fixed |
| `space-6` | 24 px | component group spacing | fixed |
| `space-8` | 32 px | card padding, compact section gap | fixed |
| `space-10` | 40 px | medium composition gap | 32–40 px |
| `space-12` | 48 px | content subsection | 40–48 px |
| `space-16` | 64 px | compact section padding | 48–64 px |
| `space-20` | 80 px | standard section padding | 56–80 px |
| `space-24` | 96 px | generous section padding | 64–96 px |
| `space-32` | 128 px | wide editorial separation | 80–128 px |
| `space-40` | 160 px | rare immersive separation | 96–160 px |

### 9.1 Spacing rules

- Use tokenized gaps rather than ad hoc margin combinations.
- Section rhythm should be generous but content-driven.
- Compact layouts reduce large spaces proportionally, not indiscriminately.
- Forms use denser spacing than editorial pages but must still feel calm.
- Touch targets and validation messages must never be compressed to preserve aesthetics.
- Adjacent components of equal hierarchy use consistent gaps.
- Background changes do not replace spacing.
