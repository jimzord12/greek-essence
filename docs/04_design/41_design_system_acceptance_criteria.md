## 41. Design-System Acceptance Criteria

The design system is implementation-ready when:

1. Primitive, semantic, and component token layers are defined and used.
2. The color palette supports all required interface states with AA contrast.
3. Cormorant Garamond and Inter are tested in English and Greek.
4. The type scale produces clear hierarchy without oversized marketing typography.
5. Spacing, containers, and responsive gutters are implemented consistently.
6. Radius, border, surface, and elevation systems are coherent.
7. Lucide icons are normalized and accessible.
8. Hero and image treatments preserve crop, meaning, and contrast.
9. Motion uses defined values and reduced-motion behavior.
10. Buttons, links, fields, choices, and statuses have complete visual states.
11. C-01 through C-30 map to implementation-ready visual rules.
12. Page types preserve the Prototype hierarchy without visual contradiction.
13. The form is clear and calm at 320 px, 390 px, 834 px, and wide desktop.
14. English and Greek remain structurally and visually equivalent.
15. Focus, contrast, target size, zoom, and forced-colors checks pass.
16. Tailwind utilities resolve from semantic CSS variables.
17. shadcn/Base UI primitives do not appear as a generic template.
18. Visual regression covers critical pages and states.
19. No design implies booking, prices, reviews, availability, or unsupported trust.
20. Open decisions remain visible rather than silently settled.
