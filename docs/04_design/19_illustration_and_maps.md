## 19. Illustration and Maps

### 19.1 Illustration

Custom illustration is not a core prototype requirement. Where used:

- keep line-based and minimal;
- use semantic palette colors;
- avoid cartoon or travel-poster style;
- ensure it adds explanatory value;
- store as optimized SVG;
- provide accessible text when meaningful.

### 19.2 Route visuals

Route summaries communicate sequence, not exact geography unless a true map is used.

- Use ordered stops and fine connecting lines.
- Pair every visual route with text.
- Horizontal route becomes vertical on compact screens.
- Route lines may use muted teal or sand.
- Active/highlighted stop uses primary color plus a non-color marker.
- Avoid fake geographic precision.

### 19.3 Maps

Maps are optional and should not become a third-party visual dependency for the prototype unless justified.

If used:

- use a restrained map style;
- keep labels legible;
- avoid interactive complexity;
- provide a textual alternative;
- do not imply live routing or availability.
