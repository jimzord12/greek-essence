
# Part VII — Iconography and Decorative Language

## 16. Iconography

**Canonical library:** Lucide.

| Context | Size | Stroke |
|---|---:|---:|
| Compact inline | 16 px | 1.75 |
| Default UI | 20 px | 1.75 |
| Large control | 24 px | 1.75 |
| Editorial feature | 24–28 px | 1.5–1.75 |

Rules:

- Icons support text; they do not replace unfamiliar labels.
- Use one icon per concept consistently.
- Decorative icons use `aria-hidden="true"`.
- Icon-only controls require accessible names and visible tooltips where necessary.
- Do not mix Lucide with another general-purpose icon family.
- Filled icons are reserved for state communication where outline-only treatment is ambiguous.
- Avoid travel clip-art such as palm trees, planes, suitcases, or map pins when a text-led treatment is more refined.
- Social and brand marks may use official SVGs, visually normalized without redrawing trademarks.
